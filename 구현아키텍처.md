# Jump Game 구현 아키텍처

## 목표

- 목표는 범용 2D 게임 엔진을 만드는 것이 아니다.
- 목표는 현재 보유한 Kenney 플랫폼er 에셋에서 실제로 쓰일 요소만 골라, 점프 게임에서 완성도 있게 동작하도록 구현하는 것이다.
- 따라서 아키텍처는 `과한 일반화`보다 `지금 필요한 기능을 안정적으로 확장하는 구조`를 우선한다.

## 주제 방향

- 메인 플레이 모드는 `슈퍼마리오형 횡스크롤 플랫폼 게임`으로 잡는다.
- 즉 기본 구조는 `자유 좌우 이동 + 점프 + 적/함정 회피 또는 처치 + 골인`에 최적화한다.
- 다만 이 구조 위에서 `점프 중심 미니게임 모드`를 파생할 수 있게 설계한다.

## 화면 기준

- 게임 기준 화면 비율은 `16:9`로 고정한다.
- 기준 해상도는 `1280 x 720`으로 잡는다.
- Windows, Web, Mobile 모두 같은 게임 월드 비율을 사용한다.
- 모바일은 세로 반응형을 따로 지원하지 않고, 가로 화면 플레이를 기본으로 한다.
- 실제 기기 비율이 다를 경우에는 게임 로직을 바꾸지 않고 스케일 또는 여백 방식으로 대응한다.

예상하는 파생 모드:

- 생존형 점프 미니게임
- 자동 스크롤 점프 챌린지
- 코인 수집 타임어택
- 적 밟기 챌린지

핵심 원칙:

- 기본 코어는 플랫폼 액션 베이스로 설계한다.
- 미니게임은 엔진을 새로 만드는 것이 아니라 `규칙`, `카메라`, `입력 제한`, `클리어 조건`을 바꾸는 방식으로 파생한다.
- 따라서 `Player`, `Collision`, `Hazard`, `Collectible`, `HUD`는 최대한 재사용 가능한 축으로 유지한다.

## 이번 프로젝트의 구현 대상

우리가 실제로 구현할 대상은 아래 범주로 제한한다.

- 플레이어 이동, 점프, 중력, 착지, 상태머신
- 지형 충돌, 일방향 발판, 경사면, 이동 발판, 떨어지는 블록
- 스프링, 사다리, 체크포인트, 문/출구
- 코인, 젬, 열쇠, 자물쇠, 하트
- 가시, 용암, 톱날, 폭탄, 낙사, 접촉 데미지, 무적 시간
- 순찰 적, 비행 적, 점프 적
- 밟기 처치, 측면 피격, 적 애니메이션, 간단한 AI 상태
- HUD, 시작 화면, 일시정지, 게임오버, 클리어
- 점프/피격/획득 사운드, 화면 흔들림, 간단한 파티클

이번 단계에서 굳이 하지 않을 것:

- 범용 이벤트 버스 대공사
- 장르 확장을 위한 추상화
- 복잡한 퀘스트/인벤토리 시스템
- 모든 적을 완전히 데이터 기반으로 생성하는 구조
- 완전 범용 타일맵 엔진

## 설계 원칙

- 플레이어, 적, 아이템, 함정은 역할 기준으로 나눈다.
- 입력 처리와 실제 이동 로직은 분리한다.
- 레벨 데이터와 게임 로직은 분리한다.
- 에셋 파일명에 게임 규칙을 직접 묶지 않는다.
- 지금 필요한 공통화만 하고, 미래를 위한 과설계는 하지 않는다.
- 나중에 바뀔 가능성이 높은 값은 상수나 설정으로 분리한다.

## 폴더별 책임

### `core`

프로젝트 전역에서 공통으로 쓰는 것을 둔다.

- 에셋 경로
- 전역 상수
- enum
- 공용 타입

예상 파일:

```text
src/core/
  assets.dart
  constants.dart
  enums.dart
  types.dart
```

### `game`

게임 전체를 조립하는 루트 계층이다.

- `JumpGame`
- 카메라 설정
- 월드 연결
- HUD 및 오버레이 연결

예상 파일:

```text
src/game/
  jump_game.dart
```

### `world`

레벨과 월드 구성을 담당한다.

- 레벨 데이터 로드
- 스폰 포인트
- 월드 상태
- 엔티티 생성 요청

예상 파일:

```text
src/world/
  level.dart
  level_loader.dart
  spawn_factory.dart
```

- 플레이어

- 수집물
- 함정
- 상호작용 오브젝트
- 발판류

예상 파일:

```text
    bee_enemy.dart

    falling_block.dart
```

### `systems`

여러 엔티티에 걸친 공통 규칙을 처리한다.

- 입력 처리
- 충돌 처리
- 데미지 처리
- 진행 처리
- 연출 처리

예상 파일:


- 적/오브젝트 설정값
- 스폰 정의

예상 파일:

```text
src/data/
```

### `ui`

## 핵심 도메인 모델

### `Actor`

움직이고 상태를 가지는 엔티티 공통 베이스다.

포함 책임:

- 위치
- 속도
- 방향
- 중력 영향
- 충돌 박스
- 상태값

하위 대상:

- `Player`
- `Enemy`
- 필요 시 일부 이동 오브젝트

### `Collectible`

획득 시 상태를 바꾸는 오브젝트다.

예시:

- 코인
- 젬
- 열쇠
- 하트

### `Hazard`

닿으면 피해를 주는 오브젝트다.

예시:

- 가시
- 용암
- 톱날
- 폭탄
- 낙사 영역

### `Interactable`

플레이어가 닿거나 조건을 만족할 때 반응하는 오브젝트다.

예시:

- 체크포인트
- 출구
- 문
- 자물쇠
- 사다리 구역
- 스프링

## 상태 설계

### 플레이어 상태

```text
idle
run
jump
fall
duck
climb
hit
dead
```

원칙:

- 입력은 상태를 바꾸는 조건일 뿐이다.
- 애니메이션은 상태의 결과다.
- 렌더링 코드가 상태를 직접 결정하지 않는다.

### 적 상태

```text
idle
patrol
jump
fly
hit
dead
```

처음부터 모든 적을 완전한 공통 AI로 묶지 않는다.  
`Enemy` 베이스를 두고 적별 업데이트 방식으로 구현한다.

## 데이터 흐름

### 입력 흐름

`Input` -> `Intent` -> `Player action/state` -> `Movement` -> `Collision resolve` -> `Animation update`

핵심은 키 입력이 플레이어 좌표를 직접 바꾸지 않게 하는 것이다.

### 레벨 로딩 흐름

`LevelDefinition` -> `SpawnFactory` -> `Entity 생성` -> `World attach`

핵심은 레벨 데이터가 `"무엇을 어디에 놓을지"`만 알고, 실제 생성은 팩토리가 맡는 구조다.

### 충돌 흐름

`Collision detected` -> `상대 역할 판별` -> `해당 시스템 또는 엔티티 처리` -> `상태 업데이트`

예:

- `Player` + `Hazard` -> 데미지 처리
- `Player` + `Collectible` -> 획득 처리
- `Player` + `Checkpoint` -> 진행 저장
- `Player` + `ExitGate` -> 클리어 처리

## 구현 우선순위

### 1단계: 기반 구조

- 폴더 구조 생성
- 에셋 상수 정리
- `JumpGame` 생성
- 기본 카메라 설정
- 빈 `Level` 구성

### 2단계: 플레이어 세로 슬라이스

- 플레이어 엔티티
- 입력 처리
- 좌우 이동
- 점프
- 중력
- 바닥 충돌
- 상태 머신
- 애니메이션 전환

이 단계 목표:

- 테스트 맵에서 플레이어가 움직이고 점프하고 착지한다.

### 3단계: 지형과 상호작용

- 기본 지형 충돌
- 일방향 발판
- 경사면
- 스프링
- 사다리
- 체크포인트
- 출구

### 4단계: 보상과 위험

- 코인
- 젬
- 열쇠
- 자물쇠
- 하트
- 가시
- 용암
- 톱날
- 낙사

### 5단계: 적 시스템

- 순찰 적
- 비행 적
- 점프 적
- 밟기 처치
- 측면 피격

### 6단계: UI와 연출

- HUD
- 시작/일시정지/게임오버/클리어

### 7단계: 데이터화와 마감

## 절대 피할 것

- `main.dart` 하나에 모든 로직 넣기
- 파일명으로 게임 규칙 분기하기
- `if level1`, `if level2` 같은 레벨 하드코딩
- 플레이어 클래스에 UI, 저장, 레벨 진행까지 몰아넣기
- 적마다 완전히 다른 구조 만들기
- 수치값을 클래스 곳곳에 흩뿌리기

## 첫 구현 대상

가장 먼저 만들 조합은 아래가 좋다.

- `JumpGame`
- `Level`
- `Player`
- `InputSystem`
- `CollisionSystem`
- `PlayerState`
- `assets.dart`
- `constants.dart`

이 조합이 잡히면 이후 코인, 함정, 적, 체크포인트를 같은 축으로 자연스럽게 붙일 수 있다.
