# 다음 진행상황

## 현재 프로젝트 한 줄 요약

`Flutter + Flame` 기반 2D 플랫폼 점프게임 프로토타입.  
현재는 `이동 -> 점프 -> 코인 수집 -> 출구 개방 -> 클리어 -> 재시작` 기본 루프 위에  
`가시/톱날 회피 -> 체크포인트 갱신 -> 리스폰 -> 스프링 점프 -> 이동 발판`까지 연결된 상태다.

## 현재 구현 완료

### 기본 게임 루프

- `GameWidget<JumpGame>` 진입 구조 연결
- 고정 해상도 카메라 구성
- HUD 표시
- 클리어 오버레이 표시
- `Retry` 시 레벨 상태와 카메라 위치 초기화

### 플레이어

- 좌우 이동
- 점프
- 중력
- 낙하 가속
- 다중 발판 착지
- 이동 발판 착지
- 스프링 점프
- 상태 전이
  - `idle`
  - `run`
  - `jump`
  - `fall`
- 좌우 방향 반전 스프라이트

### 레벨 / 월드

- 바닥 + 공중 발판 여러 개 배치
- 월드 폭 확장
- 카메라 수평 추적
- 카메라 bounds 적용
- Kenney 타일 기반 바닥 / 공중 발판 렌더링 적용
- 배경 스프라이트 적용

### 코인 / 출구

- 코인 배치 및 획득 판정
- 코인 수 HUD 반영
- 코인 애니메이션
- 모든 코인 획득 시 출구 개방
- 출구 도달 시 클리어 처리

### 위험 요소 / 리스폰

- 가시 배치 및 충돌 판정
- 톱날 배치 및 충돌 판정
- 톱날 회전 애니메이션
- 장애물 충돌 시 리스폰
- 체크포인트 활성화 시 마지막 리스폰 위치 갱신

### 상호작용 오브젝트

- 체크포인트 깃발
  - 닿으면 활성화
  - 마지막 체크포인트만 유지
- 스프링
  - 위에서 밟으면 강한 점프로 튀어오름
  - 짧은 활성화 스프라이트 전환
- 이동 발판
  - 좌우 왕복 이동
  - 위에 선 플레이어를 함께 이동시킴

### 사운드

- `flame_audio` 기반 효과음 재생 구조로 정리
- 점프 / 코인 / 클리어 / 피격 효과음 연결
- Windows에서 `just_audio_windows` 이슈를 피하도록 오디오 구조 교체 완료

## 핵심 파일

- `lib/src/game/jump_game.dart`
  - 게임 전체 흐름, HUD, 코인, 출구, 체크포인트, 스프링, 위험 요소, 리스폰 처리
- `lib/src/world/level.dart`
  - 발판, 타일 렌더링, 코인, 체크포인트, 스프링, 이동 발판, 위험 요소 배치
- `lib/src/entities/player/player.dart`
  - 이동, 점프, 중력, 착지, 이동 발판 탑승, 스프링 반응, 상태 전이
- `lib/src/entities/objects/checkpoint.dart`
  - 체크포인트 활성 / 비활성 상태와 스프라이트 관리
- `lib/src/entities/objects/spring.dart`
  - 스프링 발동과 짧은 상태 전환
- `lib/src/entities/objects/moving_platform.dart`
  - 좌우 왕복 이동 발판
- `lib/src/entities/hazards/spike.dart`
  - 가시 위험 요소
- `lib/src/entities/hazards/saw.dart`
  - 톱날 위험 요소와 회전 연출
- `lib/src/systems/audio_system.dart`
  - `flame_audio` 기반 효과음 재생 담당
- `lib/src/core/assets.dart`
  - 이미지 / 사운드 에셋 경로 정의
- `lib/src/core/constants.dart`
  - 해상도, 이동 수치, 점프 수치, 스프링 점프 수치 등 전역 상수

추천 범위:
- 하트 1~3칸
